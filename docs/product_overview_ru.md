# AI Site Analyst: краткий обзор продукта

AI Site Analyst - сервис, который помогает владельцу сайта понять, что происходит с сайтом и почему посетители не всегда превращаются в заявки.

## Для кого

- владельцы малого бизнеса;
- локальные сервисные компании;
- SEO-специалисты;
- маркетологи;
- веб-студии;
- партнеры, которые ведут сайты клиентов.

## Что делает сервис

- проверяет сайт бесплатно с главной страницы;
- позволяет зарегистрироваться и добавить сайт;
- выдает JS-код для подключения;
- собирает просмотры, клики и целевые действия;
- показывает клики по WhatsApp, телефону, email, кнопкам и формам;
- разделяет обычных посетителей и ботов;
- подключает данные Google Search Console;
- показывает AI-отчет простыми словами;
- дает оценку сайта 0-100;
- помогает понять, что исправить в первую очередь.

## Основные страницы

- `/` - бесплатная проверка сайта;
- `/login` - вход;
- `/register` - регистрация;
- `/demo` - демонстрационный режим;
- `/sites` - список сайтов пользователя;
- `/sites/new` - добавление сайта;
- `/sites/{site_id}` - обзор и аналитика сайта;
- `/settings/integrations` - статус подключений;
- `/account/password` - смена пароля;
- `/privacy`, `/terms`, `/contact` - базовые публичные страницы доверия.

## Как подключается сайт

Пользователь добавляет сайт в кабинете и получает код:

```html
<script src="https://translator-word.com/static/tracker/tracker.js" data-site-id="SITE_ID"></script>
```

Код нужно поставить перед закрывающим тегом `</body>`.

После этого сервис начинает собирать события и показывать аналитику.

## Что показывать в демо

1. Главную страницу с бесплатной проверкой.
2. Регистрацию или вход.
3. Список сайтов.
4. Добавление сайта.
5. Код подключения.
6. Обзор сайта.
7. AI-отчет.
8. Оценку сайта 0-100.
9. Демо-данные, если реальный сайт еще не подключен.

## Что проверить перед публичным запуском

- домен и HTTPS работают;
- `.env` заполнен production-значениями;
- Google OAuth настроен;
- Gemini API ключ работает и есть квота;
- PageSpeed API ключ задан, если нужен;
- SMTP подключен, если включено восстановление пароля;
- tracker.js доступен по публичному URL;
- на реальном сайте установлен правильный `data-site-id`;
- страницы `/privacy`, `/terms`, `/contact` открываются;
- ошибки 404/500 не показывают технические детали.
